## Contribution Guidelines

### Permitted Contributions

- Bug reports and issue identification
- Academic research or publication improvements
- Documentation enhancements
- Code improvements aligned with the project's scope
- Performance optimizations
- Test improvements
- Suggested features for research and academic use

### Non-Permitted Contributions

- Contributions designed for commercial use or proprietary features
- Code intended to circumvent the noncommercial license restrictions
- Features designed primarily for for-profit applications

### How to Contribute

1. **Fork the repository** and create a new branch for your work.
2. **Make your changes**, ensuring code quality and documentation.
3. **Include copyright notices** on new files: "Copyright (c) 2026 Tarek Clarke. All rights reserved."
4. **Test your changes** before submitting.
5. **Submit a pull request** with:
   - A clear description of the changes
   - Reference to any related issues
   - Confirmation that you agree to the IP assignment terms above

### Code Style

- Follow Python PEP 8 conventions
- Include docstrings for all functions and classes
- Add type hints where appropriate
- Write clear, readable code with meaningful variable names

### Licensing

All contributions must be compatible with the PolyForm Noncommercial License 1.0.0. Do not submit code with GPL, AGPL, or other copyleft licenses that would create conflicts.

## Academic and Research Use

Contributions for academic research, dissertation work, or publication are strongly encouraged. The contributor retains the right to publish their work in academic venues, provided that:

- The work is published in non-commercial academic or research contexts
- Proper attribution to the Resilient RAP Framework is included
- The project's copyright notice is preserved
